TenshiGuard AI

Project Type: Enterprise Endpoint Security Platform
Version: 1.0.0 (Live)
Date: December 2025

1. Executive Summary

TenshiGuard AI is a next-generation cybersecurity platform designed to bridge the gap between expensive enterprise EDR solutions and basic consumer antivirus software. By leveraging a hybrid approach of signature-based detection and AI-driven behavioral analysis, TenshiGuard provides real-time protection against both known malware and zero-day threats.

2. Project Agenda & Objectives

The primary agenda of this project was to architect a scalable, multi-tenant security system that could:

  • Democratize Security: Make enterprise-grade threat detection accessible to smaller organizations.
  • Automate Response: Reduce the time between detection and mitigation using AI correlation.
  • Ensure Privacy: Implement strict multi-tenancy to guarantee data isolation between organizations.

3. Problem Statement

Modern cyber threats are evolving faster than traditional defenses.

  • Legacy AV Failure: Traditional antivirus relies on known signatures, failing to stop new, polymorphic malware.
  • Alert Fatigue: Security teams are overwhelmed by thousands of false positives.
  • Fragmented Tools: Admins have to juggle separate tools for monitoring, antivirus, and network analysis.

4. Our Solution

TenshiGuard AI unifies endpoint monitoring, threat detection, and incident response into a single pane of glass.

We tackle these problems through:

  • Unified Agent: A single lightweight agent that collects telemetry, monitors files, and enforces policies.
  • AI Correlation Engine: A backend service that analyzes streams of events to identify complex attack patterns (e.g., "Brute Force" followed by "Lateral Movement").
  • Command Center: A centralized dashboard for instant visibility and remote control.

5. Technical Architecture & Technology Stack

The system is built on a robust, modern stack chosen specifically for performance, scalability, and AI capabilities.

5.1 Core Technologies

  • Python (Flask): Selected for its flexibility and extensive library support for cybersecurity and AI (Scikit-learn, Pandas). Flask's lightweight nature allows for rapid API development.
  • PostgreSQL (via Neon): Chosen for its reliability and JSONB support, allowing us to store structured relational data (Users, Devices) alongside unstructured event logs.
  • Vanilla JavaScript (ES6+): We intentionally avoided heavy frameworks like React to keep the dashboard lightweight, fast, and dependency-free, ensuring it runs smoothly even on low-resource admin consoles.
  • SQLAlchemy: Used as the ORM to provide database abstraction, ensuring secure query construction and preventing SQL injection attacks.

5.2 Infrastructure & Deployment

To ensure high availability and zero-maintenance scaling, we leverage modern serverless platforms:

  • Vercel: Hosts the application logic. Vercel's serverless architecture allows the backend to scale automatically based on traffic, ensuring the dashboard never crashes during a high-load incident.
  • Neon (Serverless Postgres): A cloud-native database that separates compute from storage. It allows us to branch databases for testing (like Git branches) and scales to zero when inactive to save costs.

6. AI Engine Deep Dive

The heart of TenshiGuard is its AI Correlation Engine, which moves beyond simple signature matching.

6.1 How It Works

The engine ingests raw telemetry streams and applies a multi-stage analysis:

  1. Event Normalization: Disparate logs (Windows Event Logs, File System changes, Network packets) are converted into a standardized JSON format.
  2. Heuristic Correlation: The engine looks for "chains" of events. For example, 5 Failed Logins (Event A) followed by 1 Successful Login (Event B) and a PowerShell Execution (Event C) triggers a "Brute Force Success" alert.
  3. Risk Scoring: Every device has a dynamic Risk Score (0-100). Critical events add +50, High +20. The score decays over time (-10% per hour) if no new threats are found, prioritizing active threats.

6.2 The End Goal

The ultimate goal of TenshiGuard AI is to build a Fully Autonomous Security Analyst. We aim to move from "Detection" to "Prediction" and "Self-Healing," where the system can patch vulnerabilities and isolate compromised endpoints before a human admin even notices the alert.

7. Key Features & Capabilities

  • Real-Time Telemetry: Live monitoring of CPU, RAM, and network usage.
  • Remote Kill Switch: Admins can isolate a device or kill a malicious process remotely.
  • Multi-Tenancy: Strict data isolation ensures Organization A never sees Organization B's data.
  • Feedback Loop: The AI learns from admin feedback (True/False Positive) to improve over time.

8. Conclusion

TenshiGuard AI represents a significant step forward in accessible automated security. By combining the speed of automated heuristics with the adaptability of Machine Learning, it provides a robust defense against the modern threat landscape.
