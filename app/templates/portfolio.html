System Architecture & Modules

TenshiGuard is not just a tool; it's an ecosystem designed for scalability, performance, and autonomy. Here are the core components I built:

Intelligent Endpoint Agent

A lightweight, Python/PowerShell-based agent that runs on client devices.

Purpose: To act as the eyes and ears on the ground, capturing telemetry that network firewalls miss.

How it works: It uses `threading` for concurrent monitoring of File Integrity (Watchdog), Process Creation (WMI), and Network Connections (Psutil), sending data every 15s.

Python PowerShell System API

AI Correlation Engine

The central brain that processes and correlates disparate events.

Purpose: To reduce alert fatigue by filtering noise and identifying complex attack patterns.

How it works: Uses heuristic chains (e.g., 5 failures + 1 success = Brute Force) and Scikit-learn for anomaly detection, assigning a dynamic risk score to each device.

Scikit-Learn Heuristics Data Pipelines

Command Center Dashboard

A responsive, real-time admin interface for visibility and control.

Purpose: To provide a "Single Pane of Glass" for security teams.

How it works: Built with Flask and Vanilla JS (no heavy frameworks) for max speed. Features live WebSocket-like polling and interactive Chart.js visualizations.

Flask Chart.js Bootstrap 5

Technology Stack & Platforms

Selected for performance, reliability, and modernize serverless capabilities.

Python 3.9+ (Flask)

Why: Unmatched libraries for Cybersec (Scapy, PyWin32) and AI (Scikit). Flask allows rapid API development.

PostgreSQL (Neon)

Why: Robust relational data structure with JSONB support for unstructured logs. Neon provides serverless scaling.

Vercel

Platform: Serverless hosting for the backend.

Why: Zero maintenance, auto-scaling, and global edge network delivery.

Cybersecurity Principles

Concepts: implemented Least Privilege, Multi-Tenancy, and Defense in Depth.

AI & The End Goal

Moving from "Detection" to "Prediction" and "Self-Healing".

How the AI Works

TenshiGuard uses a Hybrid Detection Model:

  • Signature-Based: Instant blocks for known malware hashes (like traditional AV).
  • Behavioral Analysis: It watches what a program does, not just who it is. (e.g., "Why is Notepad connecting to the internet?").
  • Risk Scoring: A dynamic score (0-100) that rises with threats and decays over time, helping admins prioritize the fire.

The Project Goal

To build a Fully Autonomous Security Analyst.

Often, human analysts are too slow to stop a ransomware encryption process (which takes milliseconds). Our goal is to create an agent smart enough to detect the anomaly and kill the process autonomously in sub-second time, alerting the human only after the threat is neutralized.
